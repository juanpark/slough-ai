### SQL Schema (PostgreSQL)

```sql
-- Workspaces table
CREATE TABLE workspaces (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    slack_team_id VARCHAR(20) UNIQUE NOT NULL,
    slack_team_name VARCHAR(255),
    decision_maker_id VARCHAR(20) NOT NULL,
    bot_token TEXT NOT NULL,
    user_token TEXT NOT NULL,
    installed_at TIMESTAMP DEFAULT NOW(),
    onboarding_completed BOOLEAN DEFAULT FALSE
);

-- Rules table
CREATE TABLE rules (
    id SERIAL PRIMARY KEY,
    workspace_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,
    rule_text TEXT NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- QA History
CREATE TABLE qa_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,
    asker_user_id VARCHAR(20) NOT NULL,
    question TEXT NOT NULL,
    answer TEXT NOT NULL,
    message_ts VARCHAR(20),
    review_status VARCHAR(20) DEFAULT 'none', -- none, requested, completed
    feedback_type VARCHAR(20), -- approved, rejected, corrected, caution
    corrected_answer TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Weekly stats
CREATE TABLE weekly_stats (
    id SERIAL PRIMARY KEY,
    workspace_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,
    week_start DATE NOT NULL,
    total_questions INT DEFAULT 0,
    review_requests INT DEFAULT 0,
    feedback_completed INT DEFAULT 0,
    UNIQUE(workspace_id, week_start)
);

-- Ingestion jobs
CREATE TABLE ingestion_jobs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,
    status VARCHAR(20) DEFAULT 'pending', -- pending, running, completed, failed
    channels_total INT DEFAULT 0,
    channels_processed INT DEFAULT 0,
    messages_processed INT DEFAULT 0,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    error_message TEXT
);
```

## Security Considerations

### Token Storage
- All Slack tokens encrypted at rest
- Use environment variables for API keys
- Never log tokens

### Data Privacy
- Decision-maker data accessible only within their workspace
- Employee questions stored with workspace isolation
- Support data deletion on uninstall

### API Security
- Verify Slack request signatures
- Rate limiting on LLM calls
- Input sanitization before LLM prompts
